Indexação por chaves

O uso de índices (ou indexação) pode ser uma alternativa à ordenação quando precisamos organizar um arquivo para ser pesquisado por chaves.


Um índice sempre aparece nas páginas finais de um livro, na forma de uma tabela com uma lista de tópicos (chaves) e os números de páginas nas quais os tópicos podem ser encontrados (campos de referência). No livro, o índice fornece uma maneira de localizar um tópico rapidamente.


Todos os índices são baseados nos mesmos conceitos básicos: chaves e campos de referência.


Especificamente para arquivos, as vantagens de utilizar índices, são :

  • Permitir impor uma ordem no arquivo sem rearranjá-lo.
  • Possibilitar múltiplas visões do arquivo criando mais de um índice.
  • Fonecer acesso rápido a arquivos com registros de tamanho variável.

Cada estrutura de índice está associada a uma chave de busca particular.


O índice primário é o fator de ordenação básico da tabela, normalmente o índice primário utiliza a chave primária. Os demais índices são chamados de índice secundário.
Quanto a técnica de construção os índices podem ser : - Índice denso : um registro de índice aparece para cada valor da chave de busca da tabela e contém o valor da chave de busca e o ponteiro para o registro. - Índice esparso : registros de índices são criados somente para alguns registros, localizando o registro com o valor menor ou igual ao desejado, depois pesquisa sequencial. Lista Invertida





De acordo com a Wikipedia, em ciência da computação, Lista Invertida (do inglês inverted list ou inverted index) é uma estrutura de dados que mapeia termos às suas ocorrências em um documento ou conjunto de documentos, armazenados em um banco de dados.


Mas o que isso significa? Vamos pensar num sistema que guarda o nome de várias pessoas e outros dados como endereço e telefone. Vamos supor que precisamos acessar estes nomes com frequência e que em determinado momento precisamos retornar todas as pessoas cujo nome é Maria. Como fazer isso sem pesquisar uma arquivo inteiro realizando vários seeks?


A solução pode ser realizada usando uma Lista Invertida. Podemos fazer um arquivo com Nomes e seus RRNs respectivos. Em outro arquivo faremos uma lista contendo a chave primária para este nome e um RRN para a próxima chave que o contém.

Árvores B e Generalizações



Uma rápida pesquisa sobre Árvores B no Google nos retorna o seguinte resultado (de acordo com a Wikipedia :D):


Se analisarmos as árvores B, elas são uma generalização das árvores binária de busca , pois cada nó de uma árvore binária armazena uma única chave de busca, enquanto as árvores B armazenam um número maior do que um de chaves de busca em cada nó, ou no termo mais usual para essa árvore, em cada página.


Em uma árvore-B, uma página, ou um nó,consiste de uma seqüência ordenada de chaves e um conjunto de ponteiros


Chamamos de ORDEM o número máximo de ponteiros que podem ser armazenados numa árvore-B. O número máximo de ponteiros é igual ao número máximo de descedentes em um nó. A árvore cresce de cima para baixo e ao chegar ao limite do número de descendentes ocorre o que chamamos de divisão e promoção. Uma árvore de ordem N, armazena N-1 chaves.

Árvore B+

A árvore B+ é derivada da árvore B, mas armazena suas chaves de maneira diferente, logo seus algoritmos de busca, inserção e remoção são diferentes. São sistemas que usam esta árovre: NTFS para o Microsoft Windows, o sistema de arquivos ReiserFS para Unix, o XFS para IRIX e Linux, e o JFS2 para AIX, OS/2 e Linux,


Principais Diferenças:

  • Todas as chaves são mantidas em folhas;
  • As chaves são repetidas em nós não-folha formando um índice;
  • As folhas são ligadas oferecendo um caminho seqüencial para percorrer as chaves.

Vantagens:

  • Mantém a eficiência da busca e da inserção da árvore B;
  • Aumenta a eficiência da localização do próximo registro na árvore de O(log2N) para O(1);
  • Não é necessário manter nenhum ponteiro de registro em nós não-folha.

Hashing (Espalhamento)

Em ciência da computação a tabela hash (de hashing, no inglês), também conhecida por tabela de espalhamento, é uma estrutura de dados especial, que associa chaves de pesquisa (hash) a valores. Seu objetivo é, a partir de uma chave simples, fazer uma busca rápida e obter o valor desejado. - WikiLivros Algoritmos e estruturas de dados


Para a maximização do desempenho de uma tabela hashing espera-se idealmente que não hajam colisões, ou seja, que cada valor atribuido leve a uma posição única na cadeia. Atribui-se portanto essa tarefa a uma função de espalhamento toda tabela de hashing é portanto gerada por uma função de espalhamento.

Essa função funciona da seguinte forma h(k) transforma uma chave k em um endereço, este endereço é usado como base para o armazenamento e recuperação de registros, isso é similar a uma indexação, pois associa a chave ao endereço relativo do registro.

Existem diversos métodos para a implementação da função de espalhamento abaixo explicitamos os dois mais populares:









Método da Divisão


Nesse método a chave K é dividida pela dimensão da tabela M, o resto da divisão é então considerado o endereço, estando sempre no intervalo entre 0 e M-1. De maneira geral acaba-se por substituir M pelo número pelo primeiro numero primo que o sucede, desde que este seja não muito próximo de potências de 2. Por exemplo: Desejamos alocar 1000 cadeias de 8 bits e podemos ter listas de tamanho 4


Temos então que M = 1000/250 = 4 Porém desejamos encontrar o primeiro número primo maior que 250


Consultando a tabela de números primos teremos que M=251 O motivo para isto está neste link.

Método do Quadrado


Esse método foi inventado por John von Neumann A chave é elevada ao quadrado e então é tirado seu modulo com M.
Exemplo:

Chave 312 -> 3122 = 97344


Considerando M igual a 100


97344 mod 100 = 44


A chave 312 foi então inserida na posição 44 da tabela de hash



unsigned int const k = 10; // M==1024
unsigned int const w = bitsizeof(unsigned int);
unsigned int h(unsigned int x) {
return (x * x) >> (w - k);
}
