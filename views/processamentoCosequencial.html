O Processamento Cosequencial consiste de operações que envolvem o processamento coordenado de duas ou mais listas de entrada sequenciais, de modo a produzir uma única lista como saída. Esse tipo de operação é muito comum em arquivos.


O objetivo desse módulo é introduzir um modelo genérico a partir do qual operações que envolvem combinações de dois ou mais arquivos de entrada possam ser implementadas.


Apesar de serem conceitualmente simples, operações que envolvem processamento cosequencial são, muitas vezes, implementadas de forma confusa e mal organizada.


Inicialmente, será apresentado um modelo genérico para a construção de procedimentos cosequenciais simples, curtos e robustos.


Exemplos de processamento cosequencial são o Merging e o Matching, o primeiro é base do processo de ordenação de arquivos muito grandes, cujo índice não cabe na memória.





Matching

Resultado
B

O objetivo desse algoritmo é produzir uma lista itens comuns das listas de entradas. Como exemplo, vamos imaginar que temos 2 arquivos que representam listas de convidados para um evento e que desejamos produzir um novo arquivo com os nomes presentes nas duas listas, considerando que as listas de entrada não contém nomes repetidos e estão na ordem crescente.


O algoritmo é basicamento o seguinte :


  1. Ler um nome de cada lista e os comparar
  2. Se ambos são iguais, copiar o nome para saída e avançar para o próximo nome nas duas listas.
  3. Se o nome da lista 1 é menor, avança na lista 1.
  4. Se o nome da lista 2 é menor, avança na lista 2.

Ao lado podemos ver o pseudo-código do algortimo.

get NAME_1 from LIST_1
get NAME_2 from LIST_2
while(MORE_NAMES_EXIST)
 if (NAME_1 < NAME_2)
  get NAME_1 from LIST_1
 else if (NAME_1 > NAME_2)
  get NAME_2 from LIST_2
 else /* match – names are the same */
  write NAME_1 to OUT_FILE
  get NAME_1 from LIST_1
  get NAME_2 from LIST_2
